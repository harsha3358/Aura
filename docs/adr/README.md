# AURA Architecture Decision Records (ADR)

This folder contains the sequential index and records of significant architectural design choices made for the AURA cognitive OS.

---

## What is an ADR?

An Architecture Decision Record (ADR) is a document that captures an important architectural decision, including its context, decisions made, alternatives considered, and consequences. 

Developers are expected to write ADRs for any breaking architectural changes, framework introductions, state management updates, or major database schema restructures.

## How to Create a New Decision Record

1. Copy the **[ADR Template](ADR_TEMPLATE.md)**.
2. Increment the ID suffix (e.g. `0002-your-decision-title.md`).
3. Add the title and details, fill out alternatives and consequences.
4. Add the new row to the table above.
5. Create a PR, obtain approval, and merge.
